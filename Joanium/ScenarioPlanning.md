Scenario planning is the discipline of thinking rigorously about uncertainty — not predicting the future, but preparing for multiple plausible versions of it so you can act confidently despite not knowing which one arrives.

## When to Use Scenario Planning

Scenario planning is most valuable when:
- The decision has significant, hard-to-reverse consequences
- There is genuine uncertainty about key variables
- The stakes of being wrong are high
- Multiple stakeholders have different assumptions about the future
- The organization needs to align on contingency responses before they're needed

**Not appropriate for:** Simple decisions with obvious answers, low-stakes reversible choices, or situations where you actually do have enough information to decide confidently without scenarios.

## Phase 1: Frame the Decision

Precisely define what you're trying to decide:
```
The focal question: "Should we [decision], and if so, how?"

Scope:
- Time horizon: What period are we planning for? (3 months? 5 years?)
- Key stakeholders: Who is affected? Who needs to be aligned?
- Success criteria: What does a good outcome look like?
- Constraints: What can't change? What must be true regardless of scenario?

Decisions to make:
1. [Primary decision]
2. [Contingent decisions that follow from it]
```

## Phase 2: Identify Key Uncertainties

Not all uncertainties are equal. Focus on the ones that would change your decision:

**Uncertainty mapping:**
```
For each relevant uncertainty, plot on two dimensions:

HIGH IMPACT + HIGH UNCERTAINTY = Scenario drivers (the ones to build scenarios around)
HIGH IMPACT + LOW UNCERTAINTY = Predetermined elements (accept as given, plan around them)
LOW IMPACT + HIGH UNCERTAINTY = Monitor but don't over-invest in planning
LOW IMPACT + LOW UNCERTAINTY = Background assumptions (set and forget)
```

**Common uncertainty categories:**
```
Market uncertainties:  demand growth, competitor moves, pricing power, regulation
Technology:           rate of adoption, disruption timing, build vs. buy outcomes
Organizational:       talent availability, execution capacity, cultural change
Financial:            funding environment, cost structure, revenue ramp
External:             macro economy, geopolitics, supply chain, regulation
```

## Phase 3: Build Scenarios

Use 2-4 scenarios. More than 4 becomes unmanageable:

**The 2×2 matrix method (most common):**
```
1. Select the two most impactful, independent uncertainties
2. Create a 2×2 matrix with high/low values on each axis
3. Each quadrant is a distinct scenario
4. Name scenarios memorably (not "Scenario A" but "The Windfall", "The Squeeze")

Example:
                    HIGH market demand
                    ┌────────────────────┐
Low competitive  │  THE GOLD RUSH     │  THE FIGHT
intensity        │  (best case)       │  (grow but compete)
                 ├────────────────────┤
High competitive │  THE NICHE         │  THE STORM
intensity        │  (survive focused) │  (worst case)
                    LOW market demand
```

**Scenario narrative structure:**
For each scenario, write a one-page description:
```
## Scenario: [Name]

**World in [timeframe]:** [2-3 sentence description of what's happened]

**Why this scenario unfolds:** [Key events/decisions that lead here]

**What this means for us:**
- Customers: [How their behavior/needs change]
- Competition: [What competitors do]
- Our business: [Revenue, cost, operational implications]
- Key risks: [What could go wrong within this scenario]
- Key opportunities: [What could we exploit]

**Early warning signals:** [What observable signs suggest this scenario is emerging?]
```

## Phase 4: Test Strategies Against Scenarios

For each strategic option, evaluate performance across scenarios:

**Strategy robustness matrix:**
```
                  SCENARIO 1   SCENARIO 2   SCENARIO 3   SCENARIO 4
Strategy A:       ★★★★★       ★★           ★★★          ★
Strategy B:       ★★★          ★★★★★      ★★           ★★★
Strategy C:       ★★★          ★★★         ★★★          ★★★
Hedged strategy:  ★★★★        ★★★★        ★★★★         ★★★

Key: ★★★★★ Thrives  ★★★ Survives  ★ Fails
```

**Three types of strategic response:**
```
1. ROBUST STRATEGIES: Perform acceptably across ALL scenarios
   Best for: core investments, organizational capabilities, brand

2. CONTINGENT STRATEGIES: Optimize for the most likely scenario, 
   with defined trigger points to pivot
   Best for: major bets, large resource allocations

3. HEDGING STRATEGIES: Reduce downside across scenarios at cost of upside
   Best for: existential risks, irreversible decisions
```

## Phase 5: Pre-Mortem Analysis

For the selected strategy, run a failure analysis BEFORE committing:

```
The Pre-Mortem Process:
1. Assume it is [timeframe] from now
2. The strategy has FAILED spectacularly
3. Write the story of how it failed — in detail
4. Go around the room; each person writes their version independently first
5. Compile all failure modes
6. Identify which failure modes are preventable
7. Build those preventions into the plan

Common failure modes to probe:
- We ran out of money before the plan worked
- A competitor moved faster and captured the market
- Our key assumption about [X] was wrong
- The team couldn't execute at the required speed/quality
- External circumstances changed in a way we didn't prepare for
- Internal misalignment caused us to execute the wrong thing well
```

## Phase 6: Decision & Monitoring System

**Make the decision:**
```
Decision record (write it down):
- Decision made: [what was decided]
- Date: 
- Decision owner:
- Rationale: [why this strategy given the scenario analysis]
- Key assumptions: [what must be true for this to work]
- Trigger conditions to revisit: [if X happens, we reconsider]
```

**Build a monitoring dashboard:**
```
For each key assumption:
Assumption: [what we believe is true]
Indicator:  [what observable signal would confirm or deny it]
Frequency:  [how often to check]
Threshold:  [at what point does this trigger a strategy review?]
Owner:      [who is responsible for monitoring this]
```

## Supporting Analytical Tools

### SWOT (for organizational context)
```
Strengths (internal advantages):    What do we do better than anyone?
Weaknesses (internal disadvantages): What are our real vulnerabilities?
Opportunities (external upside):    What trends or gaps could we exploit?
Threats (external risks):           What could hurt us externally?

Most useful insight: SWOT is only useful if it's HONEST. 
The most dangerous section is Weaknesses — it requires admitting uncomfortable truths.
```

### PESTLE (macro environment scan)
```
Political:     Government policy, political stability, regulatory changes
Economic:      Interest rates, inflation, GDP growth, employment
Social:        Demographics, cultural shifts, consumer attitudes
Technological: Emerging tech, automation risk, digital transformation
Legal:         Regulations, compliance requirements, IP landscape
Environmental: Climate impact, sustainability pressures, resource availability
```

### Force Field Analysis (for change decisions)
```
Driving forces (pushing toward the change):
→ [Force] [Strength 1-5]
→ [Force] [Strength 1-5]

Restraining forces (pushing against the change):
← [Force] [Strength 1-5]
← [Force] [Strength 1-5]

Strategy: Strengthen driving forces AND weaken restraining forces
          (most people only focus on driving forces)
```
