## Workflow

1. Start with the user task, the decision they need to make, and the state changes they will encounter.
2. Design the hierarchy, content, and interaction behavior together instead of sequentially.
3. Map empty, loading, error, retry, and expert-user states explicitly.
4. Define the handoff artifacts that engineering and QA will actually need.
5. Validate the design with critique, usability evidence, or instrumented feedback loops.
6. Capture the final rationale so later changes can be judged against original intent.

## Tradeoffs

- Expressiveness versus interface calm and clarity.
- Consistency versus local context optimization.
- Fast iteration versus handoff and validation rigor.
- Dense information versus user comprehension under stress.

## Common Failure Modes

- Using polish to hide unclear flows or weak hierarchy.
- Designing the hero state while leaving degraded states undefined.
- Handing off visuals without interaction logic or measurement intent.
- Treating critique as taste instead of a reasoning exercise.
