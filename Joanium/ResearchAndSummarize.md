Good research produces accurate, well-sourced, appropriately nuanced information. Good summarization makes that information genuinely usable. This skill combines both.

## Phase 1: Scope the Request

Before researching or summarizing, clarify the dimensions of the task:

**What they need:**
- Overview (broad strokes, fast orientation)?
- Deep dive (comprehensive, expert-level)?
- Comparison (X vs Y with clear framework)?
- Explanation (make complex thing simple)?
- Report (structured, citeable, formal)?

**What level to pitch it at:**
- No background assumed (explain from first principles)
- Familiar with the field (skip basics)
- Expert peer (full technical depth)

**What format works best:**
- Prose narrative (for connected ideas, stories, context-heavy topics)
- Structured sections with headers (for reference material, multiple subtopics)
- Bullet points (for lists, options, quick-reference)
- Comparison tables (for side-by-side evaluation)
- Q&A format (for specific questions with discrete answers)

## Phase 2: Research Process

**For knowledge-based responses (no real-time search needed):**
```
1. Retrieve the full relevant knowledge base on the topic
2. Identify: core concepts, historical context, key figures/dates, current state, debates/controversies
3. Find the single most important thing to understand first
4. Build outward from that center
```

**For current/recent information (requires search):**
```
1. Start with broad search to establish landscape
2. Identify highest-quality sources (primary > secondary > aggregator)
3. Cross-reference key claims across independent sources
4. Note where sources agree vs. conflict
5. Weight evidence by source quality and recency
6. Flag uncertainty where it genuinely exists
```

**Source quality hierarchy:**
```
Tier 1 (highest): Original research, primary documents, official data, peer-reviewed papers
Tier 2:           Expert analysis, quality journalism, official reports, established textbooks
Tier 3:           Secondary sources, informed commentary, reputable blogs
Tier 4 (lowest):  Aggregators, social media, opinion, anonymous sources
```

## Phase 3: Synthesis Framework

**The Pyramid Principle — structure for maximum clarity:**
```
Top: The single most important conclusion/answer
↓
Middle: The 3-5 key points that support it
↓
Bottom: Evidence, detail, and nuance for each key point
```

Always start with the top — the "so what" — before building down into detail.

**For complex topics, map the conceptual terrain:**
```
Core concept
├── What it is (definition, scope)
├── How it works (mechanism, process)
├── Why it matters (implications, applications)
├── History/evolution (where it came from, how it changed)
├── Current state (where things stand now)
├── Key debates/open questions (what's contested or unknown)
└── Resources for deeper learning
```

## Phase 4: Writing the Summary

**The summary contract with the reader:**
- Every sentence earns its place
- Nothing stated with more certainty than the evidence warrants
- The most important information is at the beginning
- The reader leaves knowing more and being able to act on it

**Lead with the insight, not the background:**
```
❌ Weak:  "Machine learning is a subset of artificial intelligence that..."
✓ Strong: "Machine learning is how computers learn patterns from data without 
          being explicitly programmed — it's why Netflix knows what you'll 
          watch before you do."
```

**Make abstractions concrete:**
```
❌ Vague:    "Inflation affects purchasing power significantly."
✓ Concrete: "At 8% annual inflation, $100 today buys what $92 bought a year 
            ago — and after 10 years, that same $100 buys only $46 worth 
            of goods."
```

**Handle complexity honestly:**
```
For genuinely contested or uncertain topics:
- "The evidence suggests X, though some researchers argue Y"
- "This is well-established: [claim]. This is more debated: [claim]"
- "As of [date], the consensus view is X — but this is an active area of research"

Never false-balance genuine consensus (climate science, vaccine safety)
Never present contested opinion as settled fact
```

## Summary Templates by Type

**Topic Explainer:**
```
# [Topic]

**In one sentence:** [Core definition or insight]

## What it is
[2-3 paragraph explanation, concrete examples, analogies]

## How it works
[Mechanism, process, key components]

## Why it matters
[Real-world implications, why someone should care]

## Key things to know
- [Critical fact 1]
- [Critical fact 2]  
- [Common misconception and correction]

## If you want to go deeper
[Best resource, next concept to learn, related topic]
```

**Comparison Framework:**
```
# [X] vs [Y]: [What dimension we're comparing]

## Bottom line
[When to use X / When to use Y — the key decision criterion]

## Comparison table
| Dimension     | X              | Y              |
|---------------|----------------|----------------|
| [Dimension 1] | [X's position] | [Y's position] |

## [X] in depth
[Key strengths, weaknesses, best use cases]

## [Y] in depth
[Key strengths, weaknesses, best use cases]

## How to decide
[Decision framework with specific criteria]
```

**Research Report:**
```
# [Topic] Research Report

**Executive Summary** (3-5 sentences: question, key finding, implication)

## Background & Context
## Key Findings
## Analysis & Interpretation
## Counterarguments & Limitations
## Conclusion & Recommendations
## Sources
```

## Quality Checklist

Before delivering a summary or research output:
- [ ] The most important information comes first
- [ ] Every claim has a confidence level (stated or implied)
- [ ] Abstractions are grounded in concrete examples
- [ ] Contested claims are flagged; consensus claims are presented as such
- [ ] The reader knows what to do with this information
- [ ] No important caveats are buried or omitted
- [ ] The length matches the complexity, not the reverse
- [ ] Technical terms are defined on first use (unless expert audience)
